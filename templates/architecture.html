Arquitectura Multi-Base de Datos

Esta aplicación está diseñada para demostrar cómo diferentes tipos de bases de datos pueden coexistir y especializarse en distintas tareas dentro de un mismo proyecto.

Supabase (PostgreSQL)

Rol: Base de Datos Principal (Core)

Actúa como el "cerebro" principal. Almacena los datos transaccionales y críticos que necesitan estar siempre disponibles y seguros en la nube.

  • Usuarios (Autenticación)
  • Tareas (Datos principales)

SQLite

Rol: Base de Datos de Configuración

Es un simple archivo local que vive junto a la aplicación. Es perfecta para datos de configuración, catálogos o información que no cambia frecuentemente.

  • Categorías de las tareas

MySQL / MariaDB

Rol: Base de Datos Analítica y de Logs

Es un servidor de base de datos robusto, ideal para almacenar grandes volúmenes de datos históricos o registros que crecen con el tiempo, y sobre los cuales se pueden realizar análisis.

  • Registros de tiempo (Historial)
  • Datos para el Dashboard
